## Introdução

### 🚀 **Um companheiro de desktop com inteligência artificial com possibilidades infinitas!**

#### Companheiro de Desktop: Suporta modelos VRM personalizados, ações, cenas 3D e é compatível com [extensões live2D](https://github.com/heshengtao/sap-live2d)
![image](doc/image/img-2/cover.jpeg)

#### Link VTS: Controle modelos Live2D no Vtube Studio, suportando ações personalizadas e controle de expressões
![image](doc/image/img-2/vts.jpeg)

#### Interface de chat de alta liberdade: imagens de fundo personalizadas, pacotes de emojis, personas de personagens, avatares de personagens 2D baseados em THA e ferramentas MCP de personagens para uma conversa mais imersiva
![image](doc/image/img-2/tha4.jpeg)

#### Central de Tarefas: Permita que seus agentes de IA executem qualquer tarefa avançada em segundo plano, controlem automaticamente seu computador para realizar trabalhos, com suporte a MCP e Agent Skills.
![image](doc/image/img-2/task.jpeg)

#### Controle de Computador: Através da visão do desktop combinada com uma cadeia de ferramentas para controlar o mouse, teclado e terminal, os agentes de IA podem realizar qualquer tarefa em um computador! A imagem abaixo mostra uma IA jogando Wordle.
![image](doc/image/img-2/vision.jpeg)

#### Múltiplos Modos de Janela: A interface de chat suporta encaixe no desktop em tamanho de telefone e tamanho cápsula
![image](doc/image/img-1/windowmode.jpeg)

#### Bate-papo em Grupo com Múltiplos Papéis: Suporta cartões de personagem do Tavern e memória de longo prazo, permitindo conversar com vários personagens simultaneamente!
![image](doc/image/img-2/group.jpeg)

#### Bot de Mensagens Instantâneas: Suporta implantação com um clique em QQ, WeChat, Feishu, DingTalk, Telegram, Discord, Slack
![image](doc/image/img-2/im.jpeg)

#### Bot de Live Streaming: Suporta implantação com um clique em Bilibili, YouTube, Twitch e permite transmissão ao vivo panorâmica de 360 graus.
![image](doc/image/img-2/yt.jpeg)

#### Navegador de IA: Equipa seus agentes de IA com seu próprio navegador, suportando controle automático
![image](doc/image/img-2/browser.jpeg)

#### Sistema de Extensões: Suporta a instalação de extensões e a criação de novas. A imagem abaixo mostra uma extensão galgame. Todas as extensões podem ser abertas em uma janela independente ou em uma barra lateral. Você pode até usar a skill de agente [sap-extension-creator](skills/sap-extension-creator) para criar novas extensões SAP, alcançando a auto-inicialização da agent party!
![image](doc/image/img-2/ext.jpeg)

#### Amigável para Desenvolvedores: Oferece interfaces de API OpenAI abertas e interfaces MCP, permitindo que agentes sejam conectados externamente
![image](doc/image/img-2/api.jpeg)

#### 🚀 Passos de Uso

**1. Remover a Quarentena de Download de Rede (Importante)**
Após baixar e extrair, abra o Terminal, digite o seguinte comando (observe que há um espaço no final), então arraste a **pasta extraída** para a janela do Terminal e pressione Enter:
```shell
sudo xattr -rd com.apple.quarantine 
```
*(Nota: O parâmetro `-rd` remove recursivamente os atributos de quarentena para todos os componentes dentro da pasta; caso contrário, o ambiente Python pode não funcionar corretamente.)*

**2. Conceder Permissões de Execução de Script**
Navegue até a pasta no Terminal e execute:
```shell
chmod +x 一键更新(update).sh 一键启动(start).sh
```

**3. Executar o Software**
- **Primeiro uso/Atualização:** Recomenda-se executar primeiro `./一键更新(update).sh` para garantir que as dependências estejam sincronizadas com as mais recentes.
- **Inicialização diária:** Execute diretamente `./一键启动(start).sh`.

⭐ Nota! `./super-agent-data` pode ser substituído por qualquer pasta local. Após o Docker iniciar, todos os dados serão armazenados em cache nesta pasta local e não serão carregados em nenhum outro lugar.

## Requisitos de hardware

- CPU: 2 núcleos ou mais
- Memória: 2 GB ou mais

**Como todos os modelos são opcionais, você pode acessar o mecanismo de implantação local ou usar interfaces de provedores de serviços em nuvem, portanto, há poucos requisitos de hardware. Testamos a versão Docker em um servidor de nuvem com 2 núcleos e 2 GB e funcionou perfeitamente.**

## Uso

- Desktop: Clique no ícone da área de trabalho para usar imediatamente.

- Web ou Docker: Acesse http://localhost:3456/ após a inicialização.

- Chamada de API: Amigável para desenvolvedores, perfeitamente compatível com o formato OpenAI, pode gerar saída em tempo real e não afeta a velocidade de resposta da API original. Não é necessário modificar o código de chamada:

  ```python
  from openai import OpenAI
  client = OpenAI(
    api_key="super-secret-key",
    base_url="http://localhost:3456/v1"
  )
  response = client.chat.completions.create(
    model="super-model",
    messages=[
        {"role": "user", "content": "O que é o Super Agent Party?"}
    ]
  )
  print(response.choices[0].message.content)
  ```

- Chamada MCP: Após a inicialização, você pode invocar o serviço MCP local escrevendo o seguinte conteúdo no arquivo de configuração:

  ```json
  {
    "mcpServers": {
      "super-agent-party": {
        "url": "http://127.0.0.1:3456/mcp"
      }
    }
  }
  ```

- Comandos rápidos: Na caixa de chat ou em qualquer plataforma de IM conectada (Feishu, QQ, Telegram, Discord, Slack, WeChat, WeCom, DingTalk), digite os comandos abaixo para controlar rapidamente o assistente. Você pode ativá-los/desativá-los e visualizar a lista completa em **Configurações do Sistema → Comandos Rápidos**.

  | Comando | Aliases | Escopo | Descrição |
  | --- | --- | --- | --- |
  | `/help` | `/帮助` `/?` | Chat + IM | Listar todos os comandos rápidos disponíveis |
  | `/new` | `/reset` `/重启` `/新建` `/restart` | Chat + IM | Limpar a conversa atual e começar de novo |
  | `/stop` | `/停止` | Chat + IM | Parar a resposta que está sendo gerada |
  | `/retry` | `/重试` | Chat apenas | Regenerar a última resposta |
  | `/model [provider:model]` | `/模型` | Chat + IM | Mostrar ou trocar o modelo (IM mostra apenas) |
  | `/personality [name]` | `/角色` `/persona` | Chat + IM | Mostrar ou trocar a personalidade |
  | `/skills` | `/技能` | Chat + IM | Navegar pela lista de habilidades disponíveis |
  | `/id` | - | IM apenas | Mostrar o ChatID da sessão/canal atual |
  | `/sub` | `/订阅` | IM apenas | Adicionar a sessão atual à lista de mensagens proativas |
  | `/unsub` | `/取消订阅` | IM apenas | Remover a sessão atual da lista de mensagens proativas |
  | `/plan` * | `/计划` | Chat + IM | Alternar para modo de plano (somente leitura) |
  | `/read` * | `/只读` | Chat + IM | Alternar para modo somente leitura (confirmar cada ação) |
  | `/edit` * | `/编辑` | Chat + IM | Alternar para modo de aceitar edições |
  | `/yolo` * | - | Chat + IM | Alternar para modo Yolo (autonomia total) |
  | `/cowork` * | `/协作` | Chat + IM | Alternar para modo de colaboração (sub-agentes) |
  | `/goal` * | `/目标` | Chat + IM | Alternar para modo de objetivo (loop autônomo) |
  | `/<skill-name>` * | - | Chat + IM | Injetar a habilidade especificada |
  | `#<text>` * | - | Chat + IM | Salvar memória do workspace |
  | `@<path>` * | - | Chat apenas | Referenciar um arquivo do workspace |

  > `*` Requer que o controle CLI do computador esteja ativado; injeção de habilidade, memória `#` e referência de arquivo `@` também exigem um workspace configurado.
